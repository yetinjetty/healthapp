# Test Plan – Sprint 2

To verify that the HealthMS prototype functions correctly and meets the defined functional requirements.

- Staff Approve/Reject

- Browser: Chrome

## 4. Test Approach
- Functional Testing
- Negative Testing
- Boundary Testing
- Basic Usability Testing
