Sprint 2 Testing Plan

Testing Objectives:

- To analyze feedback gathered during design validation phase.
- To incorporate feedback into prototype development.
- To ensure design validation is accurately translated into prototype development.
- To ensure interactive prototype functions according to requirements.
- To evaluate usability, user experience, and navigation flow.

Testing approach:

- Usability testing
  - Test for real-world application, ensure prototype meets user requirement
- Functional testing
  - Test for all features, ensure functionality working as intended
- Navigation Testing
  - Test for navigation flow, ensure user able to view different pages logically
- Boundary Testing
  - Test for extreme min/max value, ensure validity of limitation of inputs
- Negative Testing
  - Test for invalid inputs, ensure handling of invalid inputs and edge cases

Test Scope:

- In scope
  - Functionality - evaluate on functionality of interactive elements
  - Navigation flow - evaluate on flow of task, ensure user can logically move between screens
  - Usability - evaluate on user experience, clarity of elements
  - User Acceptance Testing (UAT) - verify that improvement from sprint 1 and prototype meets user expectations and requirements
  - Field input validation - user can input data in field in prototype
  - Backend operation - logical operations exist in prototype
  - Performance review - performance testing of prototype  

- Out of scope
  - Full system integration - only prototype design is tested, not the overall system workflow
