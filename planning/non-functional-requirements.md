Non-Functional Requirements

**A. Performance**

- NFR1 The system shall load the dashboard within 3 seconds under normal usage conditions.
- NFR2 The system shall process health data input and display confirmation within 2 seconds.
- NFR3 The system shall handle at least 100 concurrent student users without significant performance degradation.

**B. Security**

- NFR4 The system shall require authentication (login) before accessing student health data.
- NFR5 The system shall ensure that users can only access their own personal health records.
- NFR6 The system shall encrypt user passwords before storing them in the database.

**C. Usability**

- NFR7 The system interface shall be simple and intuitive, requiring no prior technical training.
- NFR8 The system shall provide clear error messages when invalid input is entered.
- NFR9 The system shall support responsive design for desktop and mobile view.

**D. Reliability**

- NFR10 The system shall maintain data consistency and prevent loss of saved health records.
- NFR11 The system shall maintain system availability of at least 95% uptime during academic semester.

**E. Maintainability**

- NFR12 The system shall follow modular design principles to allow future feature expansion.
- NFR13 The system documentation shall clearly describe system components and workflows.

**F. Ethical Considerations**

- NFR14 The system shall display a disclaimer stating that health alerts are non-diagnostic and not a medical diagnosis.
- NFR15 The system shall comply with basic data privacy principles by not sharing health data with third parties without consent.
