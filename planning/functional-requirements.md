Functional Requirements (HealthApp)

**A. Account & Profile**

- FR1 The system shall allow a student to register an account using name, email, and password.
- FR2 The system shall allow a student to log in using valid credentials.
- FR3 The system shall allow a student to update profile details (age, height, weight).

**B. Health Monitoring**

- FR4 The system shall allow a student to record health indicators (blood pressure, sleep hours, exercise frequency).
- FR5 The system shall validate health input before saving a health record (basic format/range checks).
- FR6 The system shall allow a student to view a health dashboard showing latest indicators and summary.
- FR7 The system shall allow a student to view health history (previous records by date).

**C. Reminders & Alerts**

- FR8 The system shall allow a student to receive preventive health reminders (scheduled notifications).
- FR9 The system shall generate a risk alert (non-diagnostic) when recorded values exceed defined thresholds (e.g., high BP), prompting the student to consider clinic consultation.

**D. Clinic Appointment Booking**

- FR10 The system shall allow a student to view available clinic slots based on screening schedule.
- FR11 The system shall allow a student to book an appointment by selecting a slot and providing a reason.
- FR12 The system shall allow a student to cancel an appointment (with confirmation).

**E. Clinic Staff Appointment Management**

- FR13 The system shall allow clinic staff to view appointment requests submitted by students.
- FR14 The system shall allow clinic staff to update appointment status (confirm, reject, cancel).

**F. Admin Schedule Management**

FR15 The system shall allow an admin to create screening schedules (date/time slots/capacity).
FR16 The system shall allow an admin to update screening schedules.
FR17 The system shall allow an admin to delete screening schedules.
